# Todoリストアプリ 公開手順

コードは完成済み。あとは下記を上から順にやるだけ。所要 約40分。

---

## STEP 1. スプレッドシートを用意（5分）

1. https://sheets.google.com で新規スプレッドシートを作成
2. 名前を「todoリスト」にする
3. URLの `/d/` と `/edit` の間の文字列が **スプレッドシートID**。メモする
   例: `https://docs.google.com/spreadsheets/d/【ここがID】/edit`

※ 中身（ヘッダー行など）は空のままでOK。アプリが自動で作る。

---

## STEP 2. サービスアカウントを発行（15分）

アプリがスプレッドシートを読み書きするための「ロボット用アカウント」を作る。

1. https://console.cloud.google.com にアクセス
2. 上部のプロジェクト選択 → 「新しいプロジェクト」→ 名前「todo-app」で作成
3. 左メニュー「APIとサービス」→「ライブラリ」
   → **Google Sheets API** を検索して「有効にする」
4. 左メニュー「APIとサービス」→「認証情報」
   →「認証情報を作成」→「サービスアカウント」
   → 名前「todo-bot」で作成（権限はスキップでOK）
5. 作成したサービスアカウントをクリック →「キー」タブ
   →「鍵を追加」→「新しい鍵を作成」→ **JSON** を選択 → ダウンロード
6. ダウンロードしたJSONファイルの中の `client_email`（〇〇@〇〇.iam.gserviceaccount.com）をコピー
7. STEP 1 のスプレッドシートを開き、右上「共有」
   → コピーした `client_email` を貼り付け、**編集者**権限で共有

> ✅ ダウンロードしたJSONファイルを、このフォルダに置いて俺に知らせろ。
> secrets.toml は俺が作る。

---

## STEP 3. ローカルで動作確認（任意・5分）

```bash
cd todo-app
pip install -r requirements.txt
streamlit run app.py
```

ブラウザが開いてTodoの登録・一覧・編集ができればOK。

---

## STEP 4. GitHubにpush（10分）

1. https://github.com で新規リポジトリ「todo-app」を作成（Public）
2. このフォルダをpush（コマンドは俺が出す）

> ⚠️ `.streamlit/secrets.toml` は `.gitignore` で除外済み。認証情報は絶対に上がらない。

---

## STEP 5. Streamlit Cloudで公開（5分）

1. https://share.streamlit.io にGitHubアカウントでログイン
2. 「Create app」→ リポジトリ「todo-app」、メインファイル「app.py」を選択
3. 「Advanced settings」→ Secrets欄に secrets.toml の中身を貼り付け
4. 「Deploy」→ 数分待つと公開URLが発行される 🎉

---

## 機能一覧（課題要件との対応）

- ✅ やることを登録・編集できる → 「新規登録」「編集・削除」タブ
- ✅ タイトル・内容・期日の3項目 → 登録フォームに実装
- ✅ 一覧で確認できるページ → 「一覧」タブ
- ✅ Googleスプレッドシートに保存 → gspreadで連携
- ✅ サーバーで公開 → Streamlit Community Cloud
- ➕ 完了チェック・削除機能（おまけ）
